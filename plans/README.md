## Dependency notes

- 001 must precede the merge because current public factory and indexer
  responses can expose provider credentials.
- 002 must precede visual convergence because the Lab currently prints
  success-like output for signed steward commands without executing them.
- 003 is deliberately a low-judgment import: it creates one checkout and one
  workspace without moving the imported runtime's internal directories.
- 004 waits for 003 so both Rust canisters can consume one workspace crate and
  the integration test can build both Wasms from deterministic paths.
- 005 waits for the two-canister compatibility gate from 004; otherwise an
  atomic release could atomically publish an incompatible factory/child pair.
- 006 may run after 003 once 002 is complete. It does not need to wait for the
  deployment work in 005.

## Release boundaries that must remain explicit

The final system must expose four different operations. Do not collapse them
into a generic `deploy` command:

1. Soft-deploy public playground service images without changing canisters.
2. Admit a new child artifact to the factory for future spawns.
3. Upgrade a named existing automaton canister in place.
4. Hard-reset the ephemeral playground and its state.

Publishing a monorepo release must not automatically upgrade or reinstall
existing child automatons.

## Findings considered and rejected

- Combine factory and automaton into one canister: rejected because the factory
  deliberately hands controller authority to the child itself; combining them
  would destroy that trust boundary and enlarge stable-memory blast radius.
- Replace every surface with one SPA: rejected because the certified direct
  console must remain available without the VPS stack, while evaluator controls
  are operator-only and should not enter the public Lab bundle.
- Broadly share the two schedulers or EVM state machines during the repository
  move: rejected because their domain semantics differ and the move should not
  contain unrelated behavior refactors.
- Remove per-artifact digests after adopting one Git revision: rejected because
  digests prove the exact bytes installed and allow component-level rollback.
